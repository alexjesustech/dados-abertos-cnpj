dados-abertos-cnpj — o que dá pra construir?

Pipeline já ingere 30 GB de Dados Abertos da RFB num SQLite local (68M empresas, 71M estabelecimentos, 27M sócios). Este documento mapeia o que existe no mercado, o que a lei permite, e propõe um catálogo de soluções viáveis para um projeto pessoal solo.

✅ Pipeline idempotente

Streaming dos ZIPs, retomada por Range, controle_importacao para skip. Re-execução barata.

✅ Índices essenciais

estabelecimentos por cnpj_basico, município, UF, CNAE principal; sócios por cnpj_basico.

⚠ Pontos a observar

synchronous=OFF (risco em crash OS), socios sem PK declarada, pipeline só cobre CNPJ (sem RAIS/CAGED).

2. Paisagem do mercado

Enterprise / pago

Serasa · Cortex · BigData Corp · Boa Vista

  • Score, KYB, sales intelligence, +179M empresas (Cortex)
  • Cobrança por consulta + SaaS enterprise
  • Bases proprietárias somadas (score, telefone, e-mail)

Mid-market / lead gen

Econodata · Speedio · LeadCNPJ · MonitorCNPJ

  • SaaS R$ 300–1.500/mês com filtros + export CRM
  • Enriquecimento com contato (telefone/e-mail) é o diferencial
  • MonitorCNPJ consolida 10+ portais de sanção + risco IA

Consulta unitária web

Casa dos Dados · CNPJ.biz · ConsultaCNPJ · ReceitaWS

  • SEO-driven, freemium com ads
  • API limitada (3 req/min é o padrão de free tier)
  • Casa dos Dados sob pressão LGPD recorrente

API open / freemium

CNPJ.ws · OpenCNPJ.org · minha-receita

  • minha-receita (cuducos): JSON unitário a partir dos 37 ZIPs
  • OpenCNPJ.org: free sem cap (concorrente recente)
  • CNPJ.ws Bitize: 3 req/min free → 2k req/min pago

Open source / ETL

aphonsoar (435★) · msantosjader · cuducos · okfn-brasil

  • ETLs para PostgreSQL/SQLite, alguns ativos jun/2025
  • Brasil.io (socios-brasil) com histórico de atraso
  • Espelho no BigQuery via Base dos Dados

Gaps perceptíveis

Onde quase ninguém entrega bem

  • Analytics agregado e séries históricas por setor/região
  • Grafo societário transversal grátis (Cortex/MonitorCNPJ paywall)
  • Mapas de calor por CNAE × município

3. Restrições legais — semáforo

🟢 Permitido — Dec. 8.777/2016 + LAI

  • Redistribuir os dados (atribuição à RFB)
  • Construir produto pago em cima
  • Cruzar com outras bases públicas (Compras, CEIS, BACEN, CVM, IBGE)
  • Disponibilizar APIs, dashboards e analytics agregados

🟡 Zona cinzenta — risco LGPD

  • Mostrar CPF mascarado já como vem da RFB (defensável)
  • Permitir busca reversa por nome de sócio PF (Casa dos Dados sofre pedidos LGPD recorrentes)
  • Construir grafo societário com nomes PF indexáveis
  • Perfilamento/scoring de sócio PF a partir do QSA

🔴 Evitar — risco alto

  • Reidentificar CPF mascarado cruzando com TSE/doações/servidores
  • Ligar sócio PF a processos do CNJ só por nome (falso-positivo + LGPD)
  • Usar microdado RAIS/CAGED identificado sem ACT com MTE
  • Publicar perfis pessoais derivados sem base legal sólida

Referência: STF ADIs 6387–6393 (proteção de dados ≠ uso livre só por ser público); TJDFT 2021 (Serasa, comercialização suspensa por extrapolar finalidade).

5. Catálogo de soluções possíveis

Cada card cita: dado esforço gap atendido LGPD.

A · API "minha-receita" pessoal (read-only JSON)

FastAPI / Hono na frente do SQLite. GET /cnpj/{cnpj} devolve empresa + QSA + estabelecimentos resolvidos com CNAE/município. Espelho local rápido sem rate-limit.

CNPJ only esforço P já existe (minha-receita) baixo

B · Monitor de deltas mensais (gap real)

A cada nova safra mensal, diff vs safra anterior: novo estabelecimento aberto, baixa, mudança de CNAE, mudança de QSA, mudança de capital social. Persistência em tabela eventos. Saída por feed/RSS, e-mail ou webhook por CNPJ ou por filtro (CNAE+UF).

CNPJ only esforço M ⭐ gap claro baixo (evento, não perfil)

C · Mapa de calor CNAE × município

Dashboard estático (Observable / D3 / Leaflet) com densidade de estabelecimentos por CNAE e município, séries históricas por safra. Útil para expansão de rede e jornalismo econômico.

CNPJ + IBGE esforço M ⭐ pouca oferta gratuita nenhum (agregado)

D · KYB simplificado (sanção + situação)

Dado um CNPJ, devolve: situação cadastral, sanção (CEIS/CNEP/CEPIM), contratos com a União (Compras), capital social, idade da empresa, alertas. Versão livre do MonitorCNPJ focada em transparência pública.

CNPJ + CEIS + Compras esforço M ⭐ gap (versão simples) baixo

E · Grafo de vínculos societários (PJ↔PJ)

Construir grafo apenas com sócios PJ (cnpj_socio → cnpj_empresa) usando identificador_socio = 1. Visualização interativa (Cytoscape, Neo4j-lite via sqlite-vss/grafos). Evita PF → menor risco LGPD.

CNPJ only esforço M-G ⭐ paywall hoje baixo (só PJ)

F · Lead gen B2B filtrável (CNAE + UF + idade + porte)

UI tipo "Algolia para CNPJ" com filtros, export CSV, sem telefone/e-mail (nem cruzar PF). Concorre por cima do free de Speedio/Econodata mas sem enriquecimento.

CNPJ only esforço M competitivo (vários free) médio (lista PJ tudo bem)

G · Busca semântica em razão social + atividade

Embeddings (BGE-small em PT) sobre razao_social + nome_fantasia + cnae_descricao. "Mostre padarias artesanais em Porto Velho" → CNPJs ranqueados. SQLite-vss ou Qdrant local.

CNPJ only + modelo esforço M-G ⭐ quase ninguém faz baixo

H · Dashboard econômico de Rondônia / TJRO

Recorte regional (RO) com séries históricas de abertura/baixa, top CNAEs, distribuição por município, cruzando com Compras gov.br p/ fornecedores locais da União. Encaixa no contexto institucional do autor.

CNPJ + Compras + IBGE esforço M ⭐ nicho regional vazio baixo

I · MCP server para Claude/agentes

Tool MCP sqlite-cnpj já existe (manual). Wrappar em ferramentas tipadas: buscar_empresa, vinculos_pj, cnaes_por_municipio, delta_mensal. Acelera uso pessoal e oferece como projeto open.

CNPJ only esforço P ⭐ não vi MCP equivalente baixo (uso local)

J · Validação de CNPJ alfanumérico (jul/2026)

A virada para CNPJ alfanumérico em jul/2026 ainda tem suporte inconsistente nos players. Lib + endpoint validador + reconciliação numérico↔alfanumérico vira referência rápida no ecossistema.

só regra esforço P ⭐ janela curta nenhum

6. Três caminhos sugeridos (não exclusivos)

Caminho 1 · curto prazo, alto valor de uso

"Caixa-preta de CNPJ pra mim"

A (API local) + I (MCP server) + J (validador alfanumérico). 3–4 semanas. Resolve uso pessoal/profissional, acelera Claude, e os artefatos ficam compostos.

Trade-off: não é um produto público, é uma toolchain pessoal.

Caminho 2 · projeto público diferenciado

"Observatório de deltas + grafo PJ"

B (deltas mensais) + E (grafo só PJ) + C (mapa CNAE×município). 2–3 meses. Endereça gaps reais (alertas, grafo, geo) sem entrar na zona LGPD de PF.

Trade-off: exige hosting (>30 GB) e disciplina de atualização mensal.

Caminho 3 · institucional / TJRO

"Observatório econômico de RO"

H (dashboard regional) + D (KYB simplificado para fornecedores da União em RO). 4–6 semanas. Encaixa narrativa do autor (Nugesip/TJRO), serve jornalismo local e órgãos de controle.

Trade-off: nicho regional limita reach; mas casa com contexto profissional.

7. O que não fazer

  • Replicar Casa dos Dados (busca reversa por nome de sócio PF indexada no Google) — alvo recorrente de pedidos LGPD.
  • Tentar score de crédito sem base própria — Serasa/Boa Vista têm dado restritivo proprietário, não dá pra competir só com dado aberto.
  • Vender lead gen com telefone/e-mail enriquecido — exige scraping/parceiros e cai em terreno legal/comercial saturado.
  • Re-implementar ETL CNPJ "do zero" — já tem 6+ ETLs open source maduros; o seu já roda.
